01Problem

Students don't lack tools — they lack clarity.

We talked to 23 final-year students and early-career switchers. The pattern was sharper than expected: they were drowning in resume scanners, course platforms, and LinkedIn advice — yet none of these tools answered the one question that actually mattered: “What should I do next?”

What users actually said

“My resume score is 78. Cool. Now what?”— B.Tech, 4th year
“I have 40 skills listed. I don't know which 5 actually matter for a Product role.”— Career switcher, marketing → PM
“Every course platform tells me to learn everything. I have 3 months.”— M.Sc graduate

03Solution

From score to decision.

CareerMap AI is structured around one principle: every screen must reduce a decision, not just display data. The product flows in four deliberate steps — Upload, Map, Compare, Act.

1

Upload

Resume in. Skills, projects, experience parsed.

2

Map

Pick from 7 career clusters → role-specific scoring.

3

Compare

Side-by-side roles. Reality-check vs aspiration.

4

Act

Personalised roadmap with 30/60/90-day tasks.

Design principles

  • Show the math. Every score breaks down into Skills (40%) · Projects (30%) · Experience (30%).
  • One decision per screen. No dashboards-of-everything.
  • Deterministic > clever. Rules-based engine first, LLM only where it can't hallucinate.
  • Aspiration meets reality. Always pair the 'dream role' with a 'best-fit role'.
04The Scoring Engine

Transparent by design.

We rejected the 'magic black-box score' pattern. The Readiness Score is a weighted composite users can mentally reproduce.

Readiness Score formula
Score = (Skills match × 0.40) + (Projects relevance × 0.30) + (Experience fit × 0.30)

Each sub-score is 0–100 and visible to the user with the exact contributing items.

Weight rationale

ComponentWeightWhy this weight
Skills match40%Strongest predictor of role-fit per user interviews + recruiter signal.
Projects relevance30%Proves applied ability; differentiates similar resumes.
Experience fit30%Captures domain context — but capped to avoid penalising switchers.

Edge cases we explicitly handled

  • Empty resume: Show diagnostic flow instead of a 0/100 that demoralises.
  • Over-qualified: Cap experience contribution; surface 'consider senior roles' nudge.
  • Skill spam (40+ listed): Weight only skills with project evidence.
  • Career switchers: Re-weight transferable skills using cross-cluster mapping.
05Architecture & PM Trade-offs

Choosing boring tech on purpose.

Rules-based Advisor (vs LLM)

Decision: Picked deterministic recommendations over LLM generation.

Impact: Zero hallucinations. Users trust the 'why' explanation. LLM reserved for v2 conversational layer.

Curated role taxonomy (vs scraping live job posts)

Decision: Hand-built 7 clusters × ~6 roles with explicit skill weights.

Impact: Slower to expand, but every recommendation is explainable. Live API in v2.

Client-side scoring (vs backend)

Decision: All scoring runs in the browser on parsed JSON.

Impact: Instant feedback, zero infra cost for MVP. Privacy bonus: resume never leaves device.

Compare 2 roles only (vs N)

Decision: Forced binary comparison.

Impact: Reduces analysis paralysis — the #1 problem from user research.

What I'd do differently

  • Run the cluster taxonomy past 3 recruiters before building UI — would've saved 1 week.
  • Ship the Compare flow in week 2, not week 5 — it became the most-loved feature.
  • Instrument 'explain the score back to me' as a true funnel step, not a survey question.
09Reflection

What this project taught me as a PM.

Transparency is a feature, not a footnote.

The single highest-impact decision was making the scoring formula visible. Every other piece of trust hung off it.

Decision-support beats data-display.

Users don't want more charts. They want fewer, sharper choices. We removed 4 dashboard widgets and clarity went up.

Pick the boring tech that lets you sleep.

Deterministic rules over LLMs for v1 meant zero late-night hallucination bug reports — and a product I could actually defend in user interviews.

Edge cases are the product.

Empty resumes, switchers, returners — handling these well is what separated us from 'just another scanner.'
